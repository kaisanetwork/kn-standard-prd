# PRD Review Checklist (Pre-Signoff)

## Problem and Scope

- [ ] Problem statement is evidence-based (not opinion-only)
- [ ] In-scope and out-of-scope are explicit
- [ ] Non-goals prevent scope creep

## Requirement Quality

- [ ] User stories reflect user value
- [ ] Acceptance criteria are testable and observable
- [ ] Requirements avoid implementation over-prescription

## Metrics and Outcomes

- [ ] North Star metric is defined
- [ ] Baseline + target + timeframe exist for each KPI
- [ ] At least one leading and one lagging indicator are included

## Delivery Readiness

- [ ] Dependencies are identified and owners assigned
- [ ] Key risks include mitigation plans
- [ ] Open questions have owner and due date

## Technical/Compliance

- [ ] Performance targets are quantified (e.g., p95 latency)
- [ ] Security and privacy requirements are explicit
- [ ] Accessibility expectations are specified where UI is involved

## AI Feature Add-on (if applicable)

- [ ] Evaluation method (dataset + rubric + threshold) is defined
- [ ] Hallucination/safety boundaries are defined
- [ ] Fallback behavior for low confidence is specified
